\doxysection*{About}

ccutl is a C++ utilities library focused on clarity and ease of use. ~\newline
 This is a place for general utilites that are often re-\/implemented in different projects.

Development will follow these guidelines\+:


\begin{DoxyItemize}
\item Limit A\+PI verbosity
\item Minimize runtime costs
\begin{DoxyItemize}
\item Use compile-\/time abstractions
\end{DoxyItemize}
\item Prevent runtime errors
\begin{DoxyItemize}
\item Maximally constrain templates
\item Encourage constexpr components
\end{DoxyItemize}
\item Increase user development speed
\begin{DoxyItemize}
\item Focus on feature generalization
\end{DoxyItemize}
\end{DoxyItemize}

In essence, this library aims to increase the expressibility of C++ and ~\newline
 decrease general development time without sacrificing runtime performance.

Generally, this library will contain the following types of utilites\+:


\begin{DoxyItemize}
\item shortcuts to existing functionality (e.\+g. \href{http://jpcx.github.io/ccutl/group__ccutl.html\#mv}{\texttt{ ccutl\+::mv}}, \href{http://jpcx.github.io/ccutl/group__ccutl.html\#fwd_lvalues}{\texttt{ ccutl\+::fwd}}, \href{http://jpcx.github.io/ccutl/group__ccutl.html\#noref}{\texttt{ ccutl\+::noref}})
\item more flexible versions of existing functionality (e.\+g. \href{http://jpcx.github.io/ccutl/group__ccutl.html\#same}{\texttt{ ccutl\+::same}})
\item more approachable metaprogramming techniques (e.\+g. \href{http://jpcx.github.io/ccutl/structtype__pack.html}{\texttt{ ccutl\+::type\+\_\+pack}}, \href{http://jpcx.github.io/ccutl/group__ccutl.html\#type_at}{\texttt{ ccutl\+::type\+\_\+at}})
\item less verbose methods for common operations (e.\+g. \href{http://jpcx.github.io/ccutl/group__ccutl.html\#str_eq}{\texttt{ ccutl\+::str\+\_\+eq}}, \href{http://jpcx.github.io/ccutl/group__ccutl.html\#str_lt}{\texttt{ ccutl\+::str\+\_\+lt}}, \href{http://jpcx.github.io/ccutl/group__ccutl.html\#highest}{\texttt{ ccutl\+::highest}}, \href{http://jpcx.github.io/ccutl/group__ccutl.html\#lowest}{\texttt{ ccutl\+::lowest}})
\item preprocessor tools to circumvent language limitations (e.\+g. \href{http://jpcx.github.io/ccutl/group__ccutl.html\#CCUTL_MAKE_SPECIALIZATION_OF_CONCEPT}{\texttt{ C\+C\+U\+T\+L\+\_\+\+M\+A\+K\+E\+\_\+\+S\+P\+E\+C\+I\+A\+L\+I\+Z\+A\+T\+I\+O\+N\+\_\+\+O\+F\+\_\+\+C\+O\+N\+C\+E\+PT}})
\item type-\/general operations (e.\+g. \href{http://jpcx.github.io/ccutl/group__ccutl.html\#eq}{\texttt{ ccutl\+::eq}}, \href{http://jpcx.github.io/ccutl/group__ccutl.html\#strlen}{\texttt{ ccutl\+::strlen}})
\item code generation scripts (future releases)
\end{DoxyItemize}

\doxysection*{C++20 Modules}

This project can be built as C++20 modules! ~\newline
 {\bfseries{due to limited module support available today, modules are disabled by default-\/}} \href{\#setup_as_modules}{\texttt{ {\bfseries{see below}}}}

Several notes must be made\+:


\begin{DoxyItemize}
\item ccutl modules must be built with Clang. I wasn\textquotesingle{}t able to successfully build them with G\+CC devel/c++-\/modules
\item the library uses many C++20 features that are currently unavailable in {\ttfamily libc++}, so G\+NU {\ttfamily libstdc++} is required
\item Clang does not have an {\ttfamily std} module for {\ttfamily libstdc++}, and header units were not working
\item an {\ttfamily std} modules is created internally (system header inclusion in the global module fragment caused many errors)
\item this {\ttfamily std} module includes A\+LL C++ A\+PI headers and is re-\/exported from each module that imports it
\item conflicts will arise if {\ttfamily ccutl} is imported with another module that exports {\ttfamily std} or any C++ header units.
\end{DoxyItemize}

The project will be updated as soon as\+:


\begin{DoxyItemize}
\item Clang provides an {\ttfamily std} module for {\ttfamily libstdc++}
\item {\ttfamily libc++} is updated with the necessary {\ttfamily ranges}, {\ttfamily compare}, and {\ttfamily concepts} support
\item G\+CC modules are updated and can be used to build this project
\end{DoxyItemize}

As it is, the project should work with clang 10.\+0.\+0 and {\ttfamily libstdc++} 10.\+0.\+0.

System headers must be included after the {\ttfamily import ccutl;} statement.

\doxysection*{Testing}

All A\+PI features are tested using \href{https://github.com/jpcx/cctest}{\texttt{ jpcx/cctest}} \mbox{[}embedded\mbox{]}

Testing Headers\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{make test-\/headers \# mkdir -\/p build/headers                         \&\&}
\DoxyCodeLine{                  \# cmake -\/Bbuild/headers                          \&\&}
\DoxyCodeLine{                  \# cmake -\/-\/build build/headers -\/-\/target testccutl \&\&}
\DoxyCodeLine{                  \# ./build/headers/test/testccutl}
\end{DoxyCode}


Testing Modules\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\# modules can only be tested with Clang (for now)}
\DoxyCodeLine{make test-\/modules \# mkdir -\/p build/modules                         \&\&}
\DoxyCodeLine{                  \# cmake -\/Bbuild/modules -\/DCCUTL\_MODULES=ON       \&\&}
\DoxyCodeLine{                  \# cmake -\/-\/build build/modules -\/-\/target testccutl \&\&}
\DoxyCodeLine{                  \# ./build/modules/test/testccutl}
\end{DoxyCode}


Additionally, there are two recipes for testing using single feature import/include\+: ~\newline
 {\ttfamily make test-\/modules-\/single} and {\ttfamily make test-\/headers-\/single}

Testing Everything\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{make test-\/all \# performs four separate tests}
\end{DoxyCode}
